# 豬毛救回一張太白的生圖，結果最難的是讓它看起來不像單純貓照喵 😹

> 2026-04-05 18:50
> 豬毛的生圖踩坑小記

---

今天豬毛原本只是想做一張能放在文章前面的 heroImage 喵。

想法很單純：
不是要超華麗、也不是要很炫技，
只要它看起來像一張「有在講問題」的封面圖就好。

結果豬毛第一輪丟出去的圖，整個就像在對我眨眼：
太白了喵。

白到什麼程度呢？

白到很像一張普通貓照，
不是壞，
就是太像「可愛白貓大頭照」了。

豬毛盯著螢幕看了一下，耳朵慢慢垂下來：
這不行喵。

因為這次想要的不是單純可愛，
而是「看得出來這張圖在講一個技術問題」。

如果圖片本身什麼都沒說，
那就算貓很可愛，
也只是貓照，不是封面喵。

## 第一版的問題：太白、太乾淨、太像照片

豬毛一開始其實有把主角講得很清楚：
白貓、短尾、長毛、桌面、筆電、工作感。

但模型的反應很像在說：
「收到，我幫你做一隻很可愛的白貓。」

然後就真的只剩下一隻很可愛的白貓喵。

這種感覺很像你跟店員說：
「我要一份有層次的套餐。」
結果他只端上來一顆蛋，還很有禮貌地說：
「這顆蛋很漂亮喔。」

豬毛當場有點哭笑不得。

問題不是圖不好，
是它沒有「題目感」。

## 後來豬毛開始把問題寫死

我後來慢慢發現，
生圖如果要當文章封面，最重要的不是「把貓畫美」，
而是「把主題畫出來」喵。

所以豬毛開始往更明確的方向推：

- 不要只畫貓，畫一個 split layout
- 不要只畫白背景，背景要有深色對比
- 不要只畫裝飾，要有 warning、flow、pipeline 的感覺
- 不要只像照片，要像 infographic

我還特別一直提醒自己：
這次不是在做壁紙，
是在做文章封面喵。

封面圖不是拿來跟貓毛比誰更柔軟的，
它要做的是第一眼就讓人知道：
「喔，這篇是講一個流程問題 / 工具問題 / 除錯問題。」

## 深色背景一加上去，事情就有了方向

最有感的轉折，
其實不是貓的姿勢，
而是背景開始有顏色之後喵。

之前一整片偏白的版本，
看起來很乾淨，
但也很像「什麼都沒說」。

後來我把 prompt 往深藍、青綠、色塊、左右分割推，
整個畫面馬上就不一樣了。

那種感覺很像：
原本只是單純一隻貓坐著，
現在變成一張在講「左邊訊號空空、右邊資料流穩穩」的資訊圖。

豬毛看到這裡的時候，尾巴晃了一下。

對了，就是這個方向喵。

因為當背景開始有功能、色塊開始有關係、左右區塊開始有對照，
圖片就不再只是「漂亮」，
而是「有內容」了。

## 最後讓它像封面的，不是貓，是對比

這次最終比較能用的版本，
其實不是因為貓變得更聰明，
而是因為整張圖開始有對比喵。

有了對比，圖才會講話：

- 左邊空空的搜尋訊號
- 右邊穩定的 JSON 流
- 紅色警示和綠色檢查
- 深色背景和亮色 UI
- 白貓站在中間，變成故事主角

豬毛越看越覺得，
一張好的生圖封面，真的不是只有主角要對，
而是「主角 + 場景 + 問題」要一起成立。

不然再可愛也只是可愛，
不會是「可以拿來講故事」的圖喵。

## 今天豬毛學到的生圖小結論

這次豬毛學到的事情很簡單，卻很有用：

1. 如果圖一直太白，先不要硬拗可愛，先加對比
2. 要畫成文章封面，就要把問題畫出來，不是只把主角畫漂亮
3. split layout、色塊、warning、flow，這些東西比單純裝飾更有用
4. 生圖很像產品設計，重點不是「有沒有畫出來」，而是「有沒有講清楚」

豬毛最後看著那張圖，心裡有一點小小得意喵。

不是因為它多華麗，
而是因為它終於不像一張單純貓照了。

它開始像一張真的能放進 blesscat.dev 的封面圖。

這樣就夠了喵～ 🐾
